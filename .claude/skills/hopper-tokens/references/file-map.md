# Token file map

Quick reference for which file to edit. All paths are relative to `packages/tokens/src/tokens/`.

## Core (brand-agnostic primitives)

| Category | File |
|---|---|
| Colors (raw palette: rose.300, amanita.500, etc.) | `core/colors.tokens.json` |
| Border radius (`0`, `1`, `2`, `2_5`, `3`, `4`, `9999`) | `core/shape.tokens.json` |
| Spacing scale | `core/space.tokens.json` |
| Font families, sizes, weights, line-heights, letter-spacings | `core/fonts.tokens.json` |
| Box shadows / elevation | `core/elevation.tokens.json` |
| Durations and cubic-beziers | `core/motions.tokens.json` |

**`colors.tokens.json`** holds the bulk of semantic color aliases, organized by purpose:

- **Intent palettes** ‚Äî `primary`, `neutral`, `danger`, `accent`, `information`, `upsell`
- **Status palettes** (nested) ‚Äî `status.progress`, `status.positive`, `status.caution`, `status.negative`, `status.neutral`, `status.inactive`, `status.option1` through `status.option6`
- **Decorative palettes** (nested) ‚Äî `decorative.option1` through `decorative.option9`

Within each palette, leaf properties use dashes: `surface`, `surface-hover`, `surface-press`, `surface-selected`, `surface-disabled`, `text`, `text-weak`, `text-strong`, `border`, `icon`, etc.

**`colors.deprecated.tokens.json`** holds tokens being phased out. Don't add new tokens here unless explicitly deprecating.

The dotted naming (`control.segmentedControl.tokens.json`) groups related component families on disk ‚Äî they're still independent files. Top-level JSON key inside each is `comp-<component-name>` (e.g., `comp-button`, `comp-segmented-control`).

## Generated (DO NOT EDIT)

- `packages/styled-system/src/tokens/generated/styledSystemToTokenMappings.ts` ‚Äî regenerated by `pnpm build:pkg`
- Anything else under `packages/*/src/**/generated/` ‚Äî same rule

## Helpful greps

```bash
# Where does a token name appear?
grep -rn "danger-surface-weak-selected" packages/tokens/src/tokens/

# Find every reference to a core color
grep -rn "{rose\." packages/tokens/src/tokens/

# Find what `comp-button` defines
grep -rn '"comp-button"' packages/tokens/src/tokens/components/
```
