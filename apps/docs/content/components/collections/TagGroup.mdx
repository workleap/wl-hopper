## Overview

Multiple or single tags can be used to categorize items.
Use short labels for easy scanning. Use two words only if necessary to describe the status and differentiate it from other tags.

## Best practices

Tags should:

- Be presented close to or within the input control that allows users to add and remove tags.
- Use appropriate colour semantics when describing states of an object
- Use a size that match the object it relates to. Tags should not be bigger or more prominent than the object it relates to.

<Example src="tag/docs/tag-group/disabled" />

### Disabled Item

A tag can be disabled using the `isDisabled` prop.

<Example src="tag/docs/tag-group/item/disabled" />

### Invalid Tag

If a tag group is invalid, it will display an error message. Displaying this error message will hide the helper message.

<Example src="tag/docs/tag-group/invalid" />

### Removable

Tags can be removed using the `onRemove` callback.
If you are using a keyboard, you can press the backspace key to remove a tag.

<Example src="tag/docs/tag-group/removable" />

### Readonly

Tags can be set to read-only using the `isReadOnly` prop. This will prevent the user from removing tags.

<Example src="tag/docs/tag-group/readonly" />

### Selectable

Tags can be selected using the `selectionMode` prop.
Use `defaultSelectedKeys` for initially selected items (uncontrolled) and `selectedKeys` to manage selected items (controlled).
The selected keys should match the item's `id` prop.

<Example src="tag/docs/tag-group/selectable" />
