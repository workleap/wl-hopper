# Escape Hatches - Available UNSAFE_* Props

**IMPORTANT: This is a COMPLETE list of all available UNSAFE_* props.**

- If a prop is listed here, you can use `UNSAFE_` to bypass token validation
- If a prop is NOT listed here, use it normally WITHOUT the `UNSAFE_` prefix
- NEVER use `UNSAFE_` for props that have a valid token mapping
- If a prop accepts `%`, use it normally WITHOUT the `UNSAFE_` prefix

**IMPORTANT:** NEVER use UNSAFE_ prefix with standard CSS keyword values (they're already allowed without it), such as `auto`, `inherit`, `initial`, `unset`, `fit-content`, `normal`, etc.

## When to use UNSAFE_ props

Only use `UNSAFE_` when:
1. The prop is listed below in "Available UNSAFE_* props"
2. AND you need a custom value that isn't covered by tokens
3. AND the value isn't a standard CSS keyword or percentage


## Examples

### ❌ Props that don't have UNSAFE_ versions
These props should be used normally without the `UNSAFE_` prefix:

- ❌ `UNSAFE_overflow="hidden"` - use `overflow="hidden"`
- ❌ `UNSAFE_style={...}` - use `style={...}`
- ❌ `UNSAFE_top="5px"` - use `top="5px"`
- ❌ `UNSAFE_flex="1"` - use `flex="1"`

### ❌ When tokens should be used instead
Use the appropriate token instead of UNSAFE_:

- ❌ `UNSAFE_width="32px"` - use `width="core_320"` (token available)

### ❌ Values that are already allowed
Percentages and CSS keywords work without UNSAFE_:

- ❌ `UNSAFE_height="50%"` - use `height="50%"` (percentages allowed)
- ❌ `UNSAFE_height="auto"` - use `height="auto"` (CSS keyword allowed)
- ❌ `UNSAFE_gap="normal"` - use `gap="normal"` (CSS keyword allowed)

### ✅ Correct usage
Only use UNSAFE_ for props listed below when you need custom values:

- ✅ `UNSAFE_width="100px"` - correct (width is in the list below, custom value needed)
